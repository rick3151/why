1、数据结构概念

  • 发布 2019-03-11
  • 更新 2019-03-11
  • 阅读

程序是为了具体的问题而存在的，现实生活中的问题涉及不同个体间的复杂联系，需要在计算机程序中描述生活中个体间的联系。数据结构主要研究非数值计算程序问题中的操作对象,以及它们之间的关系,而不是研究复杂的算法。

1、数据与数据元素

数据： – 程序的操作对象，用于描述客观事物 (int a, int b,) 数据是一个抽象的概念，将其进行分类后得到程序设计语言中的类型。如：int，float，char等等
数据元素：组成数据的基本单位
数据项：一个数据元素由若干数据项组成
数据对象 – 性质相同的数据元素的集合 （比如：数组，链表）

数据与数据元素、数据对象之间的关系

2、数据结构指：数据对象 中 数据元素之间的关系

数据元素之间不是独立的,存在着特定关系,这些关系即是结构。

2.1数据的逻辑结构

数据的逻辑结构：即从逻辑上描述数据之间的关系，他与数据的存储无关，是独立与计算机的.

逻辑结构分为两类：

  • 线性结构： 线性表、栈、队列、串
  • 非线性结构： 树、图（或网络）

2.2 数据的物理结构

数据的物理结构亦是存储结构，是数据的逻辑结构在计算机中存储器内的表示（或映像），它依赖于计算机。

存储结构可分为四类：顺序、链式、索引、散列

  • 顺序：借助元素在存储器中的相对位置来表示数据元素之间的逻辑关系。
  • 链式：借助指示元素存储地址的指针表示数据元素之间的关系。

  • 散列：

数据的逻辑结构与存储结构密切相关

  • 算法设计—->逻辑结构
  • 算法实现—->存储结构

3、算法

3.1算法的定义

算法是特定问题求解步骤的描述
算法是独立存在的一种解决问题的方法和思想
对于算法而言，语言不重要，重要的是思想

3.2算法和数据结构的区别

  • 数据结构是静态的描述数据元素之间的关系
  • 高效的程序需要在数据结构的基础上设计算法和选择算法
  • 程序=数据结构+算法

总结：

  • 算法是为了解决实际问题而设计的
  • 数据结构是算法需要处理的问题载体

3.3算法的特性（要素）

输入 ：算法具有0个或多个输入
输出 ：算法至少有1个或多个输出
有穷性：算法在有限的步骤之后会自动结束而不会无限循环
确定性：算法中的每一步都有确定的含义，不会出现二义性
可行性：算法的每一步都是可行的

3.4算法效率的度量

  • 事后分析法：比较运行处理时间，虽然直观但实施困难且缺陷多。
  • 多采用事前分析估算法

影响算法效率的主要因素：

  • 算法采用的策略和方法
  • 问题的输入规模
  • 编译器所产生的代码
  • 计算机执行速度

事前分析估算法：

算法最终编译成具体的计算机指令，假设每个指令的在计算机上的运行速度固定，一个算法消耗的时间也就是每条语句的执行时间之和。而循环最深处的核心语句重复执行次数也就大致决定了算法的效率执行时间。

时间复杂度： 大O表示法

操作次数的估算作为时间复杂度的度量

  • 要比较同一问题的不同算法：只需要找出基本的原操作，该基本操作重复的次数也就可以作为算法的时间度量。
  • 多数情况下它是循环最深处的语句中的原操作。

常见的时间复杂度

注解：

  • 判断一个算法的效率时，往往只需要关注操作数量的最高次项，其它次要项和常数项可以忽略。
  • 在没有特殊说明时，我们所分析的算法的时间复杂度都是指最坏时间复杂度。
  • 在判断是首先关注操作次数的最高项
  • O(5) = O(1)
  • O(2n + 1) = O(2n) = O(n)
  • O(n2+ n + 1) = O(n2)
  • O(3n3+1) = O(3n3) = O(n3)

关系比较

空间复杂度：

主要取决于程序在运行时在计算机中临时占用的存储空间大小的度量，通过计算算法的存储空间实现。也可以使用大O表示法来表示当算法执行需要的存储空间是常数时，空间复杂度为O(1)。

空间与时间的策略

  • 多数情况下，算法执行时所用的时间更令人关注
  • 如果有必要，可以通过增加空间复杂度来降低时间复杂度
  • 同理，也可以通过增加时间复杂度来降低空间复杂度

时间换空间案例

问题：在一个由自然数1-1000中某些数字所组成的数组中，每个数字可能出现零次或者多次。
设计一个算法，找出出现次数最多的数字。

  • 方法1：排序找出出现次数最多的数字
  • 方法2：开辟一个数组空间，将每个数组出现的次数缓存下来在缓存的结果中求出最大值
